# Code of Conduct

## Standard

Participants are expected to keep project discussions respectful, technical, and constructive.

Acceptable behavior includes:

- Focusing criticism on code, documentation, design, and process.
- Keeping issues and pull requests actionable.
- Respecting different development environments and experience levels.

Unacceptable behavior includes:

- Harassment, insults, or personal attacks.
- Publishing private information without permission.
- Repeatedly derailing technical discussions.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other contributions that violate this code of conduct. Maintainers may restrict participation when necessary to keep the project usable for the community.
