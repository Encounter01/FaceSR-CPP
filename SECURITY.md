# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for a suspected security vulnerability.

Report privately through the repository owner's preferred contact channel and include:

- Affected command, script, or component.
- Steps to reproduce.
- Expected impact.
- Relevant input files, paths, and logs where safe to share.

## Scope

Useful security reports include:

- Unsafe file handling.
- Path traversal or unintended file writes.
- Crashes caused by malformed image or model inputs.
- Dependency or build-chain issues that affect project users.

Model quality, dataset bias, and metric accuracy reports should usually be filed as regular issues unless they expose a concrete security risk.
